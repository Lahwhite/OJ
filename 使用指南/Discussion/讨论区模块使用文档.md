# 讨论区模块使用文档

## 1. 模块概述

讨论区模块位于 `OJ/discussion`，提供 OJ 题目讨论所需的主题、评论、回复、分页查询、Gemini AI 总结、前端页面和健康检查能力。当前版本已经接入 MySQL 持久化，并可通过独立 HTTP 服务运行。

模块主要对外提供两类入口：

- 前端页面：`GET /discussion`
- REST API：`/api/discussions/...`

Users 模块只需要在页面里增加跳转链接，将当前登录用户名传给 discussion 页面即可。

## 3. 核心组件

### 3.1 DiscussionService

`DiscussionService` 负责讨论区业务逻辑和 MySQL 读写。

主要能力：

- `createTopic`：使用 `user_id` 创建主题
- `createTopicByUsername`：使用 Users 模块的 `username` 创建主题
- `listTopics`：分页查询主题，支持按 `problem_id` 过滤
- `getTopic`：查询主题详情
- `deleteTopic`：使用 `user_id` 删除本人发表的主题
- `deleteTopicByUsername`：使用 `username` 删除本人发表的主题
- `createComment`：使用 `user_id` 创建评论
- `createCommentByUsername`：使用 `username` 创建评论
- `deleteComment`：使用 `user_id` 删除本人发表的评论或回复
- `deleteCommentByUsername`：使用 `username` 删除本人发表的评论或回复
- `listComments`：查询主题下的评论和回复
- `databaseReady`：返回讨论区依赖的 MySQL 是否可用

主题和评论返回值会关联 Users 模块的用户信息：

- `username`
- `nickname`
- `avatar`

### 3.2 DiscussionHandler

`DiscussionHandler` 负责 HTTP 路由、参数校验、JSON 响应、错误处理和 CORS。

当前已注册：

- `/discussion`：讨论区前端页面
- `/discussion/static/<path>`：前端静态资源
- `/api/discussions/topics`
- `/api/discussions/topics/{topic_id}`
- `/api/discussions/topics/{topic_id}/comments`
- `/api/discussions/topics/{topic_id}/comments/{comment_id}`
- `/api/discussions/topics/{topic_id}/summary`
- `/health`

### 3.3 GeminiClient

`GeminiClient` 负责调用 Gemini REST API，为某个主题及其评论生成中文摘要。

主要能力：

- 读取 `OJ_GEMINI_API_KEY` 或 `GEMINI_API_KEY`
- 读取 `OJ_GEMINI_MODEL`，默认使用 `gemini-2.5-flash`
- 构造讨论区总结 prompt
- 调用 `generateContent`
- 解析 Gemini 返回文本

当前 Windows 运行包使用系统 WinINet 发起 HTTPS 请求，不需要额外打包新的 HTTP 客户端 DLL。

### 3.4 前端页面

前端资源位于 `discussion/web`。页面风格与 Users 模块保持一致，使用 Tailwind CDN，主要功能包括：

- 查看主题列表
- 按题目编号筛选
- 发布主题
- 查看主题详情
- 发表评论
- 回复评论
- 删除本人发表的主题
- 删除本人发表的评论或回复
- 调用 Gemini 生成当前主题和评论的 AI 总结
- 加载更多
- 返回 Users 主页

访问示例：

```text
http://192.168.1.23:8079/discussion?username=haha
```

按题目进入：

```text
http://192.168.1.23:8079/discussion?username=haha&problem_id=1001
```

上面的 `192.168.1.23` 只是示例，请替换成 discussion 服务所在机器的局域网 IPv4。

## 4. 数据库说明

当前版本使用 MySQL 持久化，不再使用内存存储。数据库脚本位于：

```text
discussion/sql/discussion_schema.sql
```

该脚本会创建：

- `discussion_topics`
- `discussion_comments`

并依赖 Users 模块已有表：

- `Users`
- `Profiles`

讨论区通过 `Users.username` 查找用户，通过 `Profiles` 读取昵称和头像。

## 5. 配置说明

### 5.1 环境变量

| 变量 | 说明 | 默认 |
|------|------|------|
| `OJ_DISCUSSION_HOST` | 对局域网内其他机器公开的 discussion 服务 IPv4，运行包启动必填 | 无 |
| `OJ_DISCUSSION_PORT` | discussion 服务端口 | `8090`，运行包默认脚本使用 `8079` |
| `OJ_MYSQL_HOST` | MySQL 地址 | `127.0.0.1` |
| `OJ_MYSQL_PORT` | MySQL 端口 | `3306` |
| `OJ_MYSQL_USER` | MySQL 用户名 | 按环境配置 |
| `OJ_MYSQL_PASSWORD` | MySQL 密码 | 按环境配置 |
| `OJ_MYSQL_DATABASE` | MySQL 数据库 | `myOJ` |
| `OJ_REDIS_HOST` / `OJ_REDIS_PORT` | Redis 配置 | 可选 |
| `OJ_LOG_FILE` / `OJ_LOG_LEVEL` | 日志配置 | 可选 |
| `OJ_GEMINI_API_KEY` | Gemini API Key，启用 AI 总结必填 | 无 |
| `GEMINI_API_KEY` | Gemini API Key 兼容变量，`OJ_GEMINI_API_KEY` 优先 | 无 |
| `OJ_GEMINI_MODEL` | Gemini 模型 | `gemini-2.5-flash` |
| `OJ_GEMINI_HOST` | Gemini API Host | `generativelanguage.googleapis.com` |
| `OJ_GEMINI_API_VERSION` | Gemini API 版本 | `v1beta` |
| `OJ_GEMINI_TIMEOUT_MS` | Gemini 请求超时时间 | `30000` |

Redis 当前只作为公共基础设施初始化，讨论区主题和评论数据以 MySQL 为准。

## 6. 构建与运行

### 6.1 Windows + Visual Studio 构建

示例命令：

```powershell
$CONCPP = "C:\Program Files\MySQL\MySQL Connector C++ 9.7"
cmake -S OJ -B OJ\build_discussion `
  -G "Visual Studio 17 2022" -A x64 `
  -DOJ_ENABLE_MYSQL=ON `
  -DMYSQL_CONCPP_INCLUDE="$CONCPP\include\jdbc" `
  -DMYSQL_CONCPP_LIB="$CONCPP\lib64\vs14\mysqlcppconn.lib"

cmake --build OJ\build_discussion --target discussion_server --config Release
```

启动服务：

```powershell
$env:OJ_MYSQL_HOST="127.0.0.1"
$env:OJ_MYSQL_PORT="3306"
$env:OJ_MYSQL_USER="User"
$env:OJ_MYSQL_PASSWORD="Management"
$env:OJ_MYSQL_DATABASE="myOJ"
$env:OJ_DISCUSSION_PORT="8079"
$env:OJ_GEMINI_API_KEY="你的 Gemini API Key"
$env:OJ_GEMINI_MODEL="gemini-2.5-flash"

$env:PATH="C:\Program Files\MySQL\MySQL Connector C++ 9.7\lib64;$env:PATH"

.\OJ\build_discussion\discussion\Release\discussion_server.exe
```

### 6.2 使用运行包

已在 `discussion/runtime_package` 下整理 Windows 运行包，包含：

- `discussion_server.exe`
- MySQL Connector C++ 运行时 DLL
- 前端 `web/`
- 数据库脚本 `sql/`
- 启动脚本 `start_discussion.ps1`

启动：

```powershell
cd OJ\discussion\runtime_package
$env:OJ_DISCUSSION_HOST="192.168.1.23"
powershell -ExecutionPolicy Bypass -File .\start_discussion.ps1
```

运行包默认配置：

- 服务地址：`http://$env:OJ_DISCUSSION_HOST:8079/discussion`
- MySQL：`127.0.0.1:3306`
- 数据库：`myOJ`
- 用户名：`User`
- 密码：`Management`
- Gemini 模型：`gemini-2.5-flash`

局域网访问：

```text
http://你的本机局域网 IPv4:8079/discussion
```

`OJ_DISCUSSION_HOST` 需要替换成运行 discussion 服务这台机器的局域网 IPv4，不能填 `127.0.0.1`、`localhost` 或 `0.0.0.0`。如果没有设置，运行包启动脚本会直接报错。

启动脚本会自动为 `OJ_DISCUSSION_PORT` 放行 Windows 防火墙入站 TCP 端口，规则名为 `OJ Discussion 8079` 这类格式。第一次创建防火墙规则需要使用管理员 PowerShell；规则已经存在后，普通 PowerShell 启动即可。

如果要测试 AI 总结，启动前需要设置：

```powershell
$env:OJ_GEMINI_API_KEY="你的 Gemini API Key"
```

如果要修改运行包配置，可以先设置环境变量再启动：

```powershell
$env:OJ_DISCUSSION_HOST="192.168.1.23"
$env:OJ_MYSQL_HOST="127.0.0.1"
$env:OJ_MYSQL_PORT="3306"
$env:OJ_MYSQL_USER="User"
$env:OJ_MYSQL_PASSWORD="Management"
$env:OJ_MYSQL_DATABASE="myOJ"
$env:OJ_DISCUSSION_PORT="8079"
$env:OJ_GEMINI_API_KEY="你的 Gemini API Key"
$env:OJ_GEMINI_MODEL="gemini-2.5-flash"
powershell -ExecutionPolicy Bypass -File .\start_discussion.ps1
```

修改 discussion 源码或前端资源后，需要重新生成或同步运行包。原因是 `runtime_package` 中的 `discussion_server.exe` 和 `web/` 是独立拷贝，旧运行包不会自动获得新接口或新页面。

同步方式示例：

```powershell
cmake --build OJ\build_discussion --target discussion_server --config Release
Copy-Item OJ\build_discussion\discussion\Release\discussion_server.exe OJ\discussion\runtime_package\discussion_server.exe -Force
Copy-Item OJ\discussion\web\*.html OJ\discussion\runtime_package\web\ -Force
Copy-Item OJ\discussion\web\*.js OJ\discussion\runtime_package\web\ -Force
Copy-Item OJ\discussion\web\*.css OJ\discussion\runtime_package\web\ -Force
```

说明：该运行包是 Windows 版本。Linux/macOS 不能直接运行 `.exe` 和 `.dll`，需要对应平台重新构建，或改用 Docker 等跨平台交付方式。

## 7. HTTP 接口说明

### 7.1 创建主题

- URL: `/api/discussions/topics`
- Method: `POST`
- Body: JSON

字段：

- `problem_id`：必填，题目 ID
- `username`：推荐，Users 模块 session 中的登录用户名
- `user_id`：可选，若不用 `username` 则传用户 ID
- `title`：必填，最大 200
- `content`：必填，最大 5000

示例：

```json
{
  "problem_id": 1001,
  "username": "haha",
  "title": "Two Sum 求解思路",
  "content": "谁能解释 O(n) 解法？"
}
```

### 7.2 查询主题列表

- URL: `/api/discussions/topics`
- Method: `GET`

Query 参数：

- `problem_id`：可选，按题目过滤
- `limit`：可选，默认 20，最大 200
- `offset`：可选，默认 0

示例：

```text
/api/discussions/topics?problem_id=1001&limit=20&offset=0
```

### 7.3 查询主题详情

- URL: `/api/discussions/topics/{topic_id}`
- Method: `GET`

### 7.4 删除主题

- URL: `/api/discussions/topics/{topic_id}`
- Method: `DELETE`
- Body: JSON

字段：

- `username`：推荐，当前登录用户名；只能删除本人发表的主题
- `user_id`：可选，若不用 `username` 则传用户 ID

删除主题后，该主题下的评论和回复会通过数据库外键级联一起删除。

示例：

```json
{
  "username": "haha"
}
```

返回示例：

```json
{
  "topic_id": 1,
  "deleted_count": 1
}
```

### 7.5 创建评论

- URL: `/api/discussions/topics/{topic_id}/comments`
- Method: `POST`
- Body: JSON

字段：

- `username`：推荐，Users 模块 session 中的登录用户名
- `user_id`：可选，若不用 `username` 则传用户 ID
- `content`：必填，最大 3000
- `parent_comment_id`：可选，回复评论时使用

示例：

```json
{
  "username": "haha",
  "content": "建议使用哈希表。",
  "parent_comment_id": null
}
```

### 7.6 查询评论列表

- URL: `/api/discussions/topics/{topic_id}/comments`
- Method: `GET`

### 7.7 删除评论或回复

- URL: `/api/discussions/topics/{topic_id}/comments/{comment_id}`
- Method: `DELETE`
- Body: JSON

字段：

- `username`：推荐，当前登录用户名；只能删除本人发表的评论或回复
- `user_id`：可选，若不用 `username` 则传用户 ID

如果删除的是父评论，它下面的回复会随数据库外键级联一起删除；后端会同步扣减主题 `comment_count`。

示例：

```json
{
  "username": "haha"
}
```

返回示例：

```json
{
  "topic_id": 1,
  "comment_id": 2,
  "deleted_count": 3
}
```

### 7.8 Gemini 总结主题讨论

- URL: `/api/discussions/topics/{topic_id}/summary`
- Method: `POST`
- Body: 可为空 JSON，例如 `{}`

后端会读取主题正文和全部评论，调用 Gemini 生成中文摘要。没有配置 API Key 时返回 `503 gemini_not_configured`；Gemini 请求失败时返回 `502 gemini_request_failed`。

返回示例：

```json
{
  "topic_id": 1,
  "model": "gemini-2.5-flash",
  "summary": "- 讨论重点是哈希表优化。\n- 评论建议检查边界条件。",
  "comment_count": 3
}
```

### 7.9 健康检查

- URL: `/health`
- Method: `GET`

返回示例：

```json
{
  "status": "ok",
  "service": "oj-discussion",
  "mysql_pool_ready": true,
  "discussion_database_ready": true,
  "redis_ready": false,
  "gemini_configured": true,
  "gemini_model": "gemini-2.5-flash",
  "mysql_idle_connections": 1,
  "mysql_in_use_connections": 0
}
```

## 8. Users 模块集成方式

Users 模块不需要编译 discussion，也不需要安装 MySQL Connector C++。只需要增加跳转链接。

推荐在 Users 首页添加：

```html
<a th:href="@{http://192.168.1.23:8079/discussion(username=${session.loginUser}, return_url='http://127.0.0.1:8081/home')}"
   class="text-blue-400 hover:underline">讨论区</a>
```

如果从题目详情页进入，可以传 `problem_id`：

```html
<a th:href="@{http://192.168.1.23:8079/discussion(username=${session.loginUser}, problem_id=${problem.id}, return_url='http://127.0.0.1:8081/home')}">
    进入题目讨论
</a>
```

参数说明：

- `username`：当前登录用户名，discussion 用它查找 Users 表中的用户 ID
- `problem_id`：可选，进入后自动筛选某道题的讨论
- `return_url`：可选，页面“返回用户主页”按钮使用

上面的 `192.168.1.23` 只是示例，需要替换成 discussion 服务所在机器的局域网 IP，也就是启动脚本里的 `OJ_DISCUSSION_HOST`。

## 9. 测试方式

### 9.1 页面测试

启动服务后访问：

```text
http://你的本机局域网 IPv4:8079/discussion?username=haha
```

测试流程：

1. 确认页面能打开。
2. 输入题目编号、标题、内容，发布主题。
3. 点击主题列表中的主题，查看详情。
4. 发表评论。
5. 点击评论下方“回复”，提交回复。
6. 对本人发表的评论或回复点击“删除”，确认评论列表和评论数刷新。
7. 点击主题详情中的“生成总结”，确认能显示 Gemini 摘要。
8. 对本人发表的主题点击“删除主题”，确认主题从列表消失，详情区刷新。
9. 使用题目编号筛选主题。

注意：删除按钮只在当前页面 URL 的 `username` 与主题或评论作者一致时显示。例如访问 `?username=test` 时，不能删除作者为 `haha` 的主题。

### 9.2 PowerShell API 测试

```powershell
$discussionOrigin = "http://$env:OJ_DISCUSSION_HOST`:$env:OJ_DISCUSSION_PORT"

$topicBody = @{
  problem_id = 1001
  username = "haha"
  title = "Two Sum 求解思路"
  content = "谁能解释 O(n) 解法？"
} | ConvertTo-Json

$topic = Invoke-RestMethod -Method Post `
  -Uri "$discussionOrigin/api/discussions/topics" `
  -ContentType "application/json" `
  -Body $topicBody

Invoke-RestMethod "$discussionOrigin/api/discussions/topics?problem_id=1001&limit=20&offset=0"

$commentBody = @{
  username = "haha"
  content = "建议使用哈希表。"
  parent_comment_id = $null
} | ConvertTo-Json

$comment = Invoke-RestMethod -Method Post `
  -Uri ($discussionOrigin + "/api/discussions/topics/" + $topic.id + "/comments") `
  -ContentType "application/json" `
  -Body $commentBody

Invoke-RestMethod ($discussionOrigin + "/api/discussions/topics/" + $topic.id + "/comments")

Invoke-RestMethod -Method Delete `
  -Uri ($discussionOrigin + "/api/discussions/topics/" + $topic.id + "/comments/" + $comment.id) `
  -ContentType "application/json" `
  -Body (@{ username = "haha" } | ConvertTo-Json)

$env:OJ_GEMINI_API_KEY="你的 Gemini API Key"
Invoke-RestMethod -Method Post `
  -Uri ($discussionOrigin + "/api/discussions/topics/" + $topic.id + "/summary") `
  -ContentType "application/json" `
  -Body "{}"

Invoke-RestMethod -Method Delete `
  -Uri ($discussionOrigin + "/api/discussions/topics/" + $topic.id) `
  -ContentType "application/json" `
  -Body (@{ username = "haha" } | ConvertTo-Json)
```

### 9.3 CTest

普通构建下测试程序默认不会直接跑数据库集成测试。若要跑数据库测试，需确保 MySQL 配置正确并设置：


## 10. 故障排除

### 10.1 页面打不开

- 检查 discussion 服务是否启动。
- 检查端口是否正确，例如 `8079`。
- 检查是否从运行包目录启动；运行包需要能找到 `web/` 目录。

### 10.2 创建主题返回 `user_not_found`

- 确认 URL 或请求体里的 `username` 与 Users 表中的用户名一致。
- 确认 discussion 连接的是同一个 `myOJ` 数据库。

### 10.3 接口返回 `bad_request`

- 检查 JSON 是否包含必填字段。
- 检查字段类型和长度限制。

### 10.4 评论返回 `topic_not_found`

- 检查 `topic_id` 是否存在。
- 如果传了 `parent_comment_id`，确认父评论属于同一主题。

### 10.5 Windows 运行包缺 DLL

运行包中已包含当前构建所需的 MySQL Connector C++ DLL。如果换机器后仍提示缺 DLL，通常是缺 Visual C++ Runtime，需要安装 Microsoft Visual C++ Redistributable。

### 10.6 AI 总结返回 `gemini_not_configured`

- 启动 discussion 服务前设置 `OJ_GEMINI_API_KEY`。
- 如果已经启动，需要停止服务后重新启动，因为环境变量只在进程启动时读取。

### 10.7 AI 总结返回 `gemini_request_failed`

- 检查 API Key 是否有效。
- 检查当前机器是否能访问 `https://generativelanguage.googleapis.com`。
- 检查 `OJ_GEMINI_MODEL` 是否是可用模型，例如 `gemini-2.5-flash`。

## 11. 当前限制

- 认证鉴权还没有完全接入，当前前端通过 URL 参数传 `username`。
- 内容审核、编辑、点赞等高级讨论功能暂未实现。
- 评论删除当前按 `username` / `user_id` 做作者校验，后续接入统一登录态后应改为服务端会话鉴权。
- 主题删除当前也按 `username` / `user_id` 做作者校验，只有作者本人能删除。
- AI 总结每次点击会实时调用 Gemini，当前没有做数据库缓存或额度控制。
- 当前运行包是 Windows 版本，非 Windows 平台需要重新构建或使用跨平台交付方案。

## 12. 总结

当前讨论区模块已经从最初的内存版升级为可独立运行的 MySQL 持久化服务，并提供了与 Users 前端风格一致的讨论区页面和 Gemini AI 总结能力。Users 模块只需通过链接传入 `username` 和可选的 `problem_id`，即可完成页面级集成；本地联调可直接使用 `discussion/runtime_package` 下的 Windows 运行包。
