## OJ Judge 模块使用文档

1. 使用方法：通过 `judge_engine.exe --program_language=... --src_file=... --expect_result=...` 输出 JSON。

输出位置在exe相同文件夹下。

---

## 1. 环境配置

### 1.1 必备软件
- C/C++ 编译器：**MinGW g++ 13.x**
- Java
- Python3

---

## 2. 编译生成 exe 文件

在 `OJ/judge` 目录下执行：

```powershell
cd "c:\Users\33598\Desktop\se\OJ\judge"
cmake -S . -B build -G "MinGW Makefiles" -DCMAKE_CXX_COMPILER=g++
cmake -S . -B build -G "MinGW Makefiles" -DCMAKE_CXX_COMPILER=g++ -DCMAKE_EXE_LINKER_FLAGS="-static"（如果你要静态编译使用这条命令）
cmake --build build --target judge_engine --config Release
```

编译成功后可执行文件在：

`build\judge_engine.exe`

---

## 3. CLI 模式：一次性评测并输出 JSON

### 3.1 调用格式（按你要求的参数名）
```text
judge_engine.exe --program_language=<c|cpp|java|python> --src_file=<源代码文件路径> --expect_result=<期望结果json路径>
```

说明：
- `--program_language`：必须与 `config/languages.json` 中的 `id` 一致（目前支持 `c/cpp/java/python`）。
- `--src_file`：源代码文件路径。模块会直接读取文件内容作为代码。
- `--expect_result`：期望结果 JSON 文件路径（包含多组 testcases）。

### 3.2 `expect_result` JSON 格式

示例（`expect.json`）：

```json
{
  "time_limit_ms": 1000,
  "memory_limit_mb": 128,
  "test_cases": [
    {
      "id": 1,
      "input": "1 2\n",
      "expected_output": "3\n",
      "score": 10
    }
  ]
}
```

字段说明：
- `time_limit_ms`：全局时间限制（毫秒）
- `memory_limit_mb`：全局内存限制（MB）
- `test_cases[]`：
  - `id`：用例编号（也就是第几个测试用例）
  - `input`：程序的输入
  - `expected_output`：期望输出（会与实际输出做比较）
  - `score`：通过该用例获得的分数（可选；不填默认 0）

输出比较规则：实际输出与期望输出会做“去掉两端空白后的完全相等”比较。

### 3.3 CLI 返回结果（JSON）
CLI 会把评测结果 JSON 输出到 stdout。返回结构要点：

- `verdict`：最后的结果（`AC/WA/TLE/MLE/RE/CE`）
- `status`：整体状态的 int 值（与内部枚举对应）
- `runtime`：总运行时间（毫秒）
- `memory`：最大内存使用（KB）
- `total_score`：得分总和
- `max_score`：理论最大得分总和
- `test_cases`：每个用例的结果数组
  - `case_id`、`status`、`verdict`、`passed`、`runtime`、`memory`、`score`
