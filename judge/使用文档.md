# OJ Judge 模块使用文档

本文档说明如何从编译 `judge_engine` 到运行评测、查看网页结果。

---

## 1. 环境准备

### 1.1 必备软件

| 软件 | 用途 | 说明 |
|------|------|------|
| **CMake** ≥ 3.14 | 构建项目 | |
| **MinGW g++** 13.x | 编译 judge_engine | 推荐 Strawberry Perl 自带的 g++ |
| **Java (JDK)** | 评测 Java 代码 | 需 `javac`、`java` 在 PATH 中 |
| **Python 3** | 评测 Python 代码 + 启动结果页 | 需 `python` 或 `python3` 在 PATH 中 |
| **gcc / g++** | 评测 C/C++ 代码 | 与 MinGW 通常共用 |

### 1.2 检查环境

```powershell
cmake --version
g++ --version
java -version
python --version
```

## 2. 编译 judge_engine

在 `OJ/judge` 目录下执行：

```powershell
cd "c:\Users\33598\Desktop\se\OJ\judge"

# 配置（首次或 CMakeLists 变更后执行）
cmake -S . -B build -G "MinGW Makefiles" -DCMAKE_CXX_COMPILER=g++

# 编译
cmake --build build --target judge_engine
```

编译成功后，可执行文件位于：

```text
judge\build\judge_engine.exe
```

### 2.1 静态链接（可选）

若希望 exe 不依赖外部 DLL，可在配置时加链接参数：

```powershell
cmake -S . -B build -G "MinGW Makefiles" -DCMAKE_CXX_COMPILER=g++ -DCMAKE_EXE_LINKER_FLAGS="-static"
cmake --build build --target judge_engine
```

### 2.2 常见问题

| 现象 | 处理方式 |
|------|----------|
| `generator does not match` | 删除 `build` 目录后重新 `cmake -S . -B build ...` |
| 找不到 g++ | 确认 MinGW 已加入 PATH，或指定完整路径：`-DCMAKE_CXX_COMPILER=C:/Strawberry/c/bin/g++.exe` |
| 编译器与 MSVC 冲突 | 务必使用 `-G "MinGW Makefiles"`，不要用 Ninja + MSVC 混编 |

## 3. 执行评测

### 3.1 命令格式

```text
judge_engine.exe --program_language=<语言> --src_file=<源码路径> --expect_result=<期望结果JSON路径>
```

**参数说明：**

| 参数 | 必填 | 说明 |
|------|------|------|
| `--program_language` | 是 | 语言 ID：`c` / `cpp` / `java` / `python` |
| `--src_file` | 是 | 待评测源代码文件的绝对或相对路径 |
| `--expect_result` | 是 | 测试用例 JSON 文件路径 |
| `--username` | 否 | 用户名标识，默认 `admin`，用于区分不同用户的输出文件 |
| `--open_web` | 否 | 是否自动打开浏览器，默认 `1`（打开） |
| `--web_port` | 否 | 结果页端口，默认 `8080` |

### 3.2 快速示例

进入 build 目录后运行（路径按实际修改）：

```powershell
cd "c:\Users\33598\Desktop\se\OJ\judge\build"

# Java
.\judge_engine.exe `
  --program_language=java `
  --src_file=C:\Users\33598\Desktop\se\OJ\judge\tests\interface\testcode\test.java `
  --expect_result=C:\Users\33598\Desktop\se\OJ\judge\tests\interface\expect\expect0.json

# C++
.\judge_engine.exe `
  --program_language=cpp `
  --src_file=C:\Users\33598\Desktop\se\OJ\judge\tests\interface\testcode\test.cpp `
  --expect_result=C:\Users\33598\Desktop\se\OJ\judge\tests\interface\expect\expect0.json

# Python
.\judge_engine.exe `
  --program_language=python `
  --src_file=C:\Users\33598\Desktop\se\OJ\judge\tests\interface\testcode\test.py `
  --expect_result=C:\Users\33598\Desktop\se\OJ\judge\tests\interface\expect\expect0.json

# C
.\judge_engine.exe `
  --program_language=c `
  --src_file=C:\Users\33598\Desktop\se\OJ\judge\tests\interface\dp\testcode\dp_stairs.c `
  --expect_result=C:\Users\33598\Desktop\se\OJ\judge\tests\interface\dp\expect\dp_stairs.json
```

只保存 JSON、不打开浏览器：

```powershell
.\judge_engine.exe --program_language=java --src_file=... --expect_result=... --open_web=0
```

### 3.3 运行后发生什么

评测完成后，`judge_engine` 会依次：

1. **保存带时间戳的结果文件**  
   `judge\build\judge_result_<用户名>_<时间戳>.json`

2. **同步写入前端可读文件**  
   `OJ\web\latest_result.json`

3. **启动本地 HTTP 服务**（若 8080 端口尚无服务）  
   后台运行 `python -m http.server 8080`，服务目录为 `OJ\web`

4. **自动打开浏览器**  
   跳转到 `http://127.0.0.1:8080/?latest=1&t=<时间戳>`，页面自动加载并展示评测结果

终端输出示例：

```text
Result json saved to: C:\...\judge\build\judge_result_admin_1780583499283.json
Latest result for web: C:\...\OJ\web\latest_result.json
Opened result page: http://127.0.0.1:8080/?latest=1&t=1780583499283
```

## 4. 输入：expect_result JSON 格式

`--expect_result` 指向的文件描述时间/内存限制和测试用例。

### 4.1 示例

```json
{
  "time_limit_ms": 1000,
  "memory_limit_mb": 128,
  "test_cases": [
    {
      "id": 1,
      "input": "1 2\n",
      "expected_output": "3\n",
      "score": 10
    }
  ]
}
```

### 4.2 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `time_limit_ms` | int | 全局时间限制（毫秒），默认 1000 |
| `memory_limit_mb` | int | 全局内存限制（MB），默认 128 |
| `test_cases` | array | 测试用例列表（也支持别名 `testcases`） |
| `test_cases[].id` | int | 用例编号，可选，默认从 1 递增 |
| `test_cases[].input` | string | 程序 stdin 输入 |
| `test_cases[].expected_output` | string | 期望输出（也可用字段名 `output`） |
| `test_cases[].score` | int | 通过该用例得分，可选，默认 0 |

**输出比较规则：** 实际输出与期望输出在去掉两端空白后做完全相等比较。

项目内自带测试数据：

```text
judge/tests/interface/expect/expect0.json          # 简单 A+B
judge/tests/interface/dp/expect/dp_stairs.json     # 多组 DP 用例
```

## 5. 输出：评测结果 JSON 格式

### 5.1 成功时

```json
{
  "program_language": "java",
  "overall_status": 2,
  "verdict": "AC",
  "runtime": 42,
  "memory": 2048,
  "error_message": "",
  "total_score": 10,
  "max_score": 10,
  "test_cases": [
    {
      "case_id": 1,
      "status": 2,
      "verdict": "AC",
      "passed": true,
      "runtime": 42,
      "memory": 2048,
      "score": 10
    }
  ]
}
```

### 5.2 字段说明

| 字段 | 说明 |
|------|------|
| `verdict` | 总判定：`AC` / `WA` / `TLE` / `MLE` / `RE` / `CE` |
| `overall_status` | 状态整型枚举（0=PENDING, 1=JUDGING, 2=AC, 3=WA, …） |
| `runtime` | 总运行时间（毫秒） |
| `memory` | 峰值内存（KB） |
| `total_score` / `max_score` | 实际得分 / 满分 |
| `error_message` | 编译错误等信息（CE 时有内容） |
| `test_cases[]` | 每个用例的 `case_id`、`verdict`、`passed`、`runtime`、`memory`、`score` |

### 5.3 判定含义

| verdict | 含义 |
|---------|------|
| AC | Accepted，答案正确 |
| WA | Wrong Answer，输出不匹配 |
| TLE | Time Limit Exceeded，超时 |
| MLE | Memory Limit Exceeded，内存超限 |
| RE | Runtime Error，运行时错误 |
| CE | Compile Error，编译失败 |

### 5.4 CLI 错误时

```json
{
  "error": "cli_error",
  "message": "具体错误描述"
}
```

## 6. 网页结果展示

前端位于 `OJ/web/`，与 judge 联动使用。

### 6.1 自动模式（推荐）

直接运行 `judge_engine.exe`，评测结束后浏览器会自动打开结果页，无需手动操作。

### 6.2 手动模式

**方式 A：手动启动 HTTP 服务**

```powershell
cd "c:\Users\33598\Desktop\se\OJ\web"
python -m http.server 8080
```

浏览器访问 <http://127.0.0.1:8080/>，拖入 `judge_result_*.json` 或粘贴 JSON 后点击「解析并展示」。

**方式 B：直接打开 HTML**

双击 `web/index.html`，通过拖放或粘贴 JSON 查看（不依赖 HTTP 服务，但无法使用「示例数据」按钮）。

### 6.3 页面展示内容

- 总判定（大字号 AC/WA/…）
- 得分进度、总用时、峰值内存、通过用例数
- 每个测试用例的状态色块与明细表
- 可展开查看原始 JSON

## 7. 完整流程一览

```text
1. 安装 CMake、MinGW、Java、Python
        ↓
2. cmake 配置 + 编译 judge_engine.exe
        ↓
3. 准备源码文件 + expect_result JSON
        ↓
4. 运行 judge_engine.exe（三个必填参数）
        ↓
5. 自动保存 JSON → 自动打开浏览器展示结果
```

## 8. 故障排查

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| `Failed to open language config file` | 在 `build/` 下运行，找不到 `config/languages.json` | 不影响评测，会使用内置默认配置；或在 `judge/` 目录运行 exe |
| 浏览器未打开 | Python 未安装或 `--open_web=0` | 安装 Python；或手动打开 `web/index.html` 导入 JSON |
| 页面空白 / 加载失败 | 8080 端口被占用或防火墙拦截 | 换端口：`--web_port=8888`；或手动 `python -m http.server 8888` |
| Java 评测 CE | JDK 未加入 PATH | 确认 `javac`、`java` 可用 |
| C/C++ 评测 CE | gcc/g++ 未加入 PATH | 确认 MinGW 的 bin 在 PATH 中 |
