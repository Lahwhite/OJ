# Judge + Web 改动文档

本文档记录为实现「单 exe 交付、端口自动避让、无 Python 依赖」而对 `judge/` 与 `web/` 所做的全部改动。

---

## 1. 改动背景

| 原有问题 | 目标 |
|----------|------|
| 端口默认写死 8080 | 自动扫描空闲端口，避免与他人服务冲突 |
| 主机写死 127.0.0.1 | 可通过参数 / 配置文件 / 环境变量修改 |
| 依赖 Python 启动静态页 | exe 内置 HTTP 服务 + 内嵌前端资源 |
| 依赖磁盘上的 `OJ/web/` 目录 | 编译期嵌入 `web/`，单 exe 即可展示结果 |
| 只有一种展示方式 | 支持 `embedded` / `file_only` / `external` 三种模式 |

---

### judge/

| 文件 | 说明 |

| `config/judge_cli.json` | 默认运行时配置（端口 auto、模式 embedded 等） |

### web/

| 文件 | 说明 |
|------|------|
| `config.example.json` | 交付方可复制的配置示例 |

## 4. 架构变化

### 4.1 之前

```text
judge_engine.exe
  → 写 judge_result_*.json + web/latest_result.json
  → python -m http.server 8080
  → 打开 http://127.0.0.1:8080/?latest=1
```

### 4.2 现在

```text
judge_engine.exe（评测模式）
  → 写 judge_result_<timestamp>.json
  → 解析配置（CLI / 环境变量 / config/judge_cli.json）
  → [embedded] 后台启动 judge_engine.exe --serve_web=<port>（内嵌 HTTP + 内嵌页面）
  → 打开 http://<web_host>:<port>/?latest=1&t=<ts>

judge_engine.exe（--serve_web 内部模式）
  → 在指定端口提供 index.html / css / js / latest_result.json
  → 运行 web_keep_alive_sec 秒后自动退出
```

---

## 5. 配置体系

### 5.1 优先级

```text
命令行参数  >  环境变量  >  config/judge_cli.json  >  程序默认值
```

### 5.2 新增参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--web_mode` | `embedded` | `embedded` / `file_only` / `external` |
| `--web_port` | `auto` | `auto` 或具体端口号 |
| `--web_host` | `127.0.0.1` | HTTP 绑定主机 |
| `--web_port_range` | `8080-8180` | auto 模式扫描范围 |
| `--web_keep_alive_sec` | `300` | 内嵌 HTTP 服务存活秒数 |
| `--open_web` | `1` | 是否自动打开浏览器 |
| `--web_url` | 空 | external 模式下的外部页面地址 |
| `--config` | 自动查找 | 指定配置文件路径 |

### 5.3 环境变量

| 变量 | 对应配置 |
|------|----------|
| `OJ_WEB_MODE` | web_mode |
| `OJ_WEB_PORT` | web_port |
| `OJ_WEB_HOST` | web_host |
| `OJ_WEB_PORT_RANGE` | web_port_range（如 `8080-8180`） |
| `OJ_WEB_URL` | web_url |
| `OJ_OPEN_WEB` | open_web |
| `OJ_WEB_KEEP_ALIVE_SEC` | web_keep_alive_sec |

### 5.4 三种 web_mode

| 模式 | 行为 |
|------|------|
| `embedded` | 内嵌 HTTP + 内嵌页面，自动选端口，打开浏览器 |
| `file_only` | 仅写 JSON，不占端口，不打开浏览器 |
| `external` | 仅写 JSON，打开 `--web_url` 指定的已有前端 |

---

## 6. 标准输出约定

评测结束后，stdout 会输出便于脚本解析的行：

```text
Result json saved to: <path>
OJ_RESULT_JSON=<path>
OJ_WEB_URL=http://127.0.0.1:8093/?latest=1&t=...
OJ_WEB_PORT=8093
```

（`file_only` 模式下无 `OJ_WEB_URL` / `OJ_WEB_PORT`。）

## 7. 构建变化

```powershell
cd judge
cmake -S . -B build -G "MinGW Makefiles" -DCMAKE_CXX_COMPILER=g++
cmake --build build --target judge_engine
```

构建时会自动执行：

```text
python tools/embed_web_assets.py ../web src/web_assets.cpp
```

**编译依赖：** CMake、MinGW g++、Python 3（仅构建阶段用于资源嵌入，运行时不需要）。

## 8. 兼容性说明

| 项目 | 说明 |
|------|------|
| 旧命令格式 | 完全兼容，`--program_language` / `--src_file` / `--expect_result` 不变 |
| 旧 JSON 输出格式 | 不变 |
| `web/latest_result.json` | **不再写入**；结果仅保存在 exe 同目录的 `judge_result_*.json`，由内置 HTTP 按需读取 |
| Python HTTP 服务 | **已移除**，不再需要 |
| 独立 `web/` 目录 | 开发时仍保留；交付单 exe 时可不带 |

---

## 9. 已知限制

1. 内置 HTTP 服务当前仅实现 Windows（Winsock）；非 Windows 平台可正常评测，但 `embedded` 模式暂无法自动展示页面。
2. `external` 模式假定外部前端已实现 `?latest=1` 或自行处理 JSON。
3. 每次评测会启动一个短时 HTTP 子进程，默认 300 秒后自动退出；连续评测可能占用多个端口（在 configured range 内递增）。

---

## 10. 回滚方式

若需回到旧行为（Python + 固定 8080 + 写 web/latest_result.json），可 checkout 改动前的 `judge/src/main.cpp` 并移除新增源文件。当前版本已不建议在生产交付中使用该方式。
