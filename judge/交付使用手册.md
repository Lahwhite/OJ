# OJ Judge 交付使用手册

本手册面向**收到交付包的使用者**，说明如何在任意一台 Windows 电脑上完成评测并查看结果。

---

## 1. 交付物说明

典型交付包结构：

```text
OJ-Judge/
├── judge_engine.exe          # 主程序（含评测引擎 + 结果页，单文件即可运行展示）
├── config/
│   └── judge_cli.json        # 可选，Web 展示相关配置
└── 交付使用手册.md           # 本文档
```

**最小交付：** 仅 `judge_engine.exe` 一个文件即可运行评测；若需自定义端口等行为，可附带 `config/judge_cli.json`。

---

## 2. 目标电脑环境要求

### 2.1 必需

| 项目 | 用途 |
|------|------|
| **Windows 10 / 11（64 位）** | 运行 judge_engine.exe |
| **对应语言的编译/运行环境** | 见下表 |

### 2.2 按评测语言准备

| 语言 | 需要安装 | 验证命令 |
|------|----------|----------|
| **C** | MinGW gcc | `gcc --version` |
| **C++** | MinGW g++ | `g++ --version` |
| **Java** | JDK（含 javac、java） | `java -version` 和 `javac -version` |
| **Python** | Python 3 | `python --version` |

> **注意：** 交付包**不需要**安装 Python 来查看结果页；Python 仅在你要评测 Python 代码时才需要。

### 2.3 将编译器加入 PATH

安装 MinGW / JDK / Python 后，请确认在 **PowerShell** 或 **cmd** 中能直接运行上表中的命令。若提示「找不到命令」，需将安装目录下的 `bin` 加入系统环境变量 PATH。

## 3. 快速开始（3 步）

### 第 1 步：准备文件

- **源代码文件**，例如 `Main.java`
- **测试用例 JSON**（`expect_result`），例如 `cases.json`：

```json
{
  "time_limit_ms": 1000,
  "memory_limit_mb": 128,
  "test_cases": [
    {
      "id": 1,
      "input": "1 2\n",
      "expected_output": "3\n",
      "score": 10
    }
  ]
}
```

### 第 2 步：打开终端

按 `Win + R`，输入 `powershell`，进入存放 `judge_engine.exe` 的目录，例如：

```powershell
cd "D:\OJ-Judge"
```

### 第 3 步：运行评测

```powershell
.\judge_engine.exe `
  --program_language=java `
  --src_file="D:\path\to\Main.java" `
  --expect_result="D:\path\to\cases.json"
```

**成功时：**

1. 终端显示 `Result json saved to: ...`
2. 浏览器自动打开结果页，展示 AC/WA、得分、各测试用例详情
3. exe 同目录生成 `judge_result_<时间戳>.json`

## 4. 命令参数完整说明

### 4.1 必填参数

```text
--program_language=<c|cpp|java|python>
--src_file=<源代码文件路径>
--expect_result=<测试用例JSON路径>
```

路径建议使用**绝对路径**，或相对于当前工作目录的路径；含空格时请用引号包裹。

### 4.2 可选参数（Web 展示）

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--open_web=1` | 1 | `0` = 不打开浏览器 |
| `--web_mode=embedded` | embedded | 见下文三种模式 |
| `--web_port=auto` | auto | `auto` 自动选空闲端口；或指定如 `9000` |
| `--web_host=127.0.0.1` | 127.0.0.1 | 仅本机访问，一般无需修改 |
| `--web_port_range=8080-8180` | 8080-8180 | auto 模式下的端口扫描范围 |
| `--web_keep_alive_sec=300` | 300 | 结果页 HTTP 服务存活时间（秒） |
| `--web_url=` | 空 | external 模式使用 |
| `--config=` | 自动 | 指定配置文件路径 |

### 4.3 三种 Web 模式

#### embedded（默认，推荐）

- 自动在空闲端口启动内嵌结果页
- 自动打开浏览器
- **适合：** 普通用户、本地开发、演示

#### file_only（脚本 / 服务器）

```powershell
.\judge_engine.exe ... --web_mode=file_only
```

- 只生成 JSON，不占端口，不打开浏览器
- **适合：** CI 流水线、批处理脚本、无图形界面服务器

#### external（对接已有 OJ 前端）

```powershell
.\judge_engine.exe ... --web_mode=external --web_url="http://192.168.1.100:8080/result/"
```

- 只写 JSON，浏览器打开指定的外部地址
- **适合：** 已有统一部署的结果展示页面

## 5. 配置文件（可选）

将 `config/judge_cli.json` 放在 `judge_engine.exe` 同目录的 `config/` 下，或在 exe 同目录直接放 `judge_cli.json`：

```json
{
  "open_web": true,
  "web_mode": "embedded",
  "web_host": "127.0.0.1",
  "web_port": "auto",
  "web_port_range": [8080, 8180],
  "web_keep_alive_sec": 300,
  "web_url": ""
}
```

**示例：端口常被占用时**

```json
{
  "web_port": "auto",
  "web_port_range": [9000, 9100]
}
```

**示例：服务器上只输出 JSON**

```json
{
  "open_web": false,
  "web_mode": "file_only"
}
```

也可通过环境变量覆盖（适合批量部署）：

```powershell
$env:OJ_WEB_MODE = "file_only"
$env:OJ_OPEN_WEB = "0"
.\judge_engine.exe --program_language=cpp --src_file=... --expect_result=...
```

## 7. 输出结果说明

### 7.1 终端输出

```text
Result json saved to: D:\OJ-Judge\judge_result_1780590263228.json
OJ_RESULT_JSON=D:\OJ-Judge\judge_result_1780590263228.json
Opened result page: http://127.0.0.1:8093/?latest=1&t=1780590263228
OJ_WEB_URL=http://127.0.0.1:8093/?latest=1&t=1780590263228
OJ_WEB_PORT=8093
```

脚本可通过 `OJ_RESULT_JSON=` 行获取 JSON 文件路径。

### 7.2 判定含义

| 显示 | 含义 |
|------|------|
| **AC** | 通过 |
| **WA** | 答案错误 |
| **TLE** | 超时 |
| **MLE** | 内存超限 |
| **RE** | 运行时错误 |
| **CE** | 编译失败 |

### 7.3 浏览器未自动打开时

1. 查看终端中的 `OJ_WEB_URL=` 行，复制地址到浏览器手动打开  
2. 或直接打开 `judge_result_*.json` 文件查看原始数据  
3. 若不需要浏览器，使用 `--open_web=0` 或 `--web_mode=file_only`

## 8. 常见问题

### Q1：提示找不到 gcc / g++ / java / python

**原因：** 对应语言运行环境未安装或未加入 PATH。  
**解决：** 安装 MinGW（C/C++）、JDK（Java）或 Python 3，并将 `bin` 目录加入系统 PATH，重新打开终端后再试。

### Q2：浏览器打开但页面空白 / 加载失败

**原因：** 端口被防火墙拦截，或 HTTP 子进程未成功启动。  
**解决：**

```powershell
# 指定其他端口范围
.\judge_engine.exe ... --web_port=auto --web_port_range=9000-9100
```

或改用 `--web_mode=file_only`，直接查看 JSON 文件。

### Q3：8080 端口已被占用

**无需处理。** 默认 `web_port=auto` 会在 8080–8180 范围内自动寻找空闲端口；终端会打印实际使用的 `OJ_WEB_PORT`。

### Q4：在服务器上运行，不想弹浏览器

```powershell
.\judge_engine.exe ... --web_mode=file_only
```

### Q5：杀毒软件拦截 judge_engine.exe

内嵌 HTTP 服务会短暂监听本地端口，部分安全软件可能提示。**允许本地网络访问**即可；该服务仅绑定 `127.0.0.1`，不对外网开放。

### Q6：评测 CE（编译错误）

查看终端同目录下的 `judge_result_*.json` 中 `error_message` 字段，或结果页上的错误信息。

## 9. 批处理脚本示例

```powershell
# run_judge.ps1
param(
  [Parameter(Mandatory=$true)][string]$Lang,
  [Parameter(Mandatory=$true)][string]$Source,
  [Parameter(Mandatory=$true)][string]$Expect
)

$exe = Join-Path $PSScriptRoot "judge_engine.exe"
& $exe `
  --program_language=$Lang `
  --src_file=$Source `
  --expect_result=$Expect `
  --web_mode=embedded `
  --web_port=auto

if ($LASTEXITCODE -ne 0) {
  Write-Error "Judge failed with exit code $LASTEXITCODE"
  exit $LASTEXITCODE
}
```

用法：

```powershell
.\run_judge.ps1 -Lang java -Source "C:\work\Main.java" -Expect "C:\work\cases.json"
```

## 10. 检查清单（交付验收）

在目标电脑上依次确认：

- [ ] `judge_engine.exe` 能双击或命令行启动（无缺少 DLL 报错）
- [ ] 目标语言编译器在 PATH 中可用
- [ ] 运行一条完整评测命令，终端出现 `OJ_RESULT_JSON=`
- [ ] 浏览器自动展示结果（embedded 模式）
- [ ] exe 同目录生成 `judge_result_*.json`
- [ ] `--web_mode=file_only` 时不弹浏览器、仍生成 JSON
- [ ] 本机 8080 已被占用时，仍能自动换端口并成功打开页面

全部通过即表示交付物可在该环境正常运行。

## 11. 技术支持信息

| 项目 | 说明 |
|------|------|
| 主程序 | `judge_engine.exe` |
| 结果 JSON | exe 同目录 `judge_result_<时间戳>.json` |
| 默认 Web 模式 | embedded（内嵌页面 + 自动端口） |
| 默认端口策略 | auto，范围 8080–8180 |
| 运行时是否需要 Python | **否**（仅评测 Python 代码时需要 Python） |
| 是否需要单独 web 文件夹 | **否**（已编译进 exe） |

更详细的开发侧改动说明见同目录下的 `改动文档.md`。
