# OJ 题目管理模块实现说明

## 1. 模块范围

题目管理模块负责题目发布、题目维护、分类搜索、测试用例维护，以及给评测/排行榜模块提供基础对接接口。

已覆盖接口：

- `GET /api/v1/problems`：公开题目分页查询，支持难度、标签、关键词筛选
- `GET /api/v1/problems/{id}`：公开题目详情查询，只返回样例测试点
- `GET /api/v1/problems/{id}/test-cases`：获取题目完整测试用例，供评测模块使用
- `GET /api/v1/problems/tags`：获取标签列表，供前端筛选使用
- `POST /api/v1/problems/{id}/submissions`：评测结束后回写提交统计，供排行榜/题单展示使用
- `POST /api/v1/problems`：管理员创建题目
- `PUT /api/v1/problems/{id}`：管理员更新题目
- `DELETE /api/v1/problems/{id}`：管理员删除题目

## 3. 模块说明

- Controller 负责接口层、参数校验和鉴权入口。
- Service 负责题目、标签、测试用例的一致性维护。
- Repository 负责访问 `problems`、`test_cases`、`tags`、`problem_tags`。
- 管理员接口要求 `Authorization: Bearer <JWT_TOKEN>`，且 JWT payload 中 `role=admin`。
- 公开题目详情接口只返回样例测试点，避免普通前端直接拿到隐藏测试数据。
- 完整测试用例接口保留给评测模块对接，课程项目阶段暂不额外加内部服务密钥。
- 提交统计接口用于评测模块在判题完成后更新 `submission_count` 和 `accepted_count`。

JWT payload 可采用：

```json
{
  "user_id": 1,
  "role": "admin"
}
```

## 5. 接口简表

### 5.1 查询题目列表

```http
GET /api/v1/problems?page=1&size=20&difficulty=easy&tag=数组&keyword=两数
```

响应核心字段：

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "total": 1,
    "page": 1,
    "size": 20,
    "problems": [
      {
        "id": 1,
        "title": "两数之和",
        "difficulty": "easy",
        "tags": ["数组", "哈希表"],
        "passRate": 0.8,
        "submissionCount": 10,
        "acceptedCount": 8
      }
    ]
  }
}
```

### 5.2 查询题目详情

```http
GET /api/v1/problems/1
```

说明：`testCases` 只包含 `isSample=true` 的样例测试点。

### 5.3 获取完整测试用例

```http
GET /api/v1/problems/1/test-cases
```

说明：供评测模块按题目 ID 拉取完整输入输出数据。

### 5.4 查询标签列表

```http
GET /api/v1/problems/tags
```

### 5.5 回写提交统计

```http
POST /api/v1/problems/1/submissions
Content-Type: application/json

{
  "accepted": true
}
```

规则：

- 每调用一次，`submission_count + 1`。
- 当 `accepted=true` 时，`accepted_count + 1`。
- 私有或不存在题目返回 `404002`。

### 5.6 创建/更新题目

```http
POST /api/v1/problems
Authorization: Bearer <admin-token>
Content-Type: application/json

{
  "title": "两数之和",
  "description": "给定两个整数，输出它们的和。",
  "difficulty": "easy",
  "timeLimit": 1000,
  "memoryLimit": 128,
  "inputDescription": "输入两个整数。",
  "outputDescription": "输出它们的和。",
  "sampleInput": "1 2",
  "sampleOutput": "3",
  "tags": ["数组", "入门"],
  "isPublic": true,
  "testCases": [
    {
      "input": "1 2",
      "output": "3",
      "isSample": true,
      "score": 20
    },
    {
      "input": "10 20",
      "output": "30",
      "isSample": false,
      "score": 80
    }
  ]
}
```

更新题目使用 `PUT /api/v1/problems/{id}`，请求体同上。

### 5.7 删除题目

```http
DELETE /api/v1/problems/1
Authorization: Bearer <admin-token>
```

## 6. 数据库脚本

- `users`

- `test_cases`
- `tags`

## 7. 运行方式

修改 `management/src/main/resources/application.yml` 中的数据库和 JWT 配置后，在 `management` 目录执行：

```bash
mvn spring-boot:run
```

或打包运行：

```bash
mvn clean package
java -jar target/problem-management-1.0.0.jar
```

## 8. 测试方式

在 `management` 目录执行：

```bash
mvn test
```

测试覆盖：

- `ProblemControllerTest`：接口响应、参数校验、管理员鉴权、新增标签和提交统计接口
- `ProblemServiceImplTest`：分页映射、详情样例过滤、完整测试用例、标签维护、提交统计、删除更新异常
- `ProblemServiceIntegrationTest`：基于 H2 的创建、查询、更新统计、更新题目、删除题目完整流程
- `JwtTokenServiceTest`：JWT 签名校验和权限校验
- `GlobalExceptionHandlerTest`：统一错误响应
- `ApiResponseTest`：统一响应体

## 9. 对其他模块的对接建议

- 前端模块：
  - 题库页调用 `GET /api/v1/problems`。
  - 标签筛选调用 `GET /api/v1/problems/tags`。
  - 题目详情页调用 `GET /api/v1/problems/{id}`。
  - 管理员后台调用创建、更新、删除接口。
- 评测模块：
  - 用户提交代码时，根据 `problemId` 调用 `GET /api/v1/problems/{id}/test-cases` 获取完整测试用例。
  - 判题结束后调用 `POST /api/v1/problems/{id}/submissions` 回写是否 AC。
- 排行榜模块：
  - 可直接读取题目统计字段，或通过题目列表/详情接口获取 `submissionCount`、`acceptedCount`、`passRate`。

## 10. 当前取舍

这是课程项目实现，优先保证鲁棒性和对接完整性：

- 未引入 Swagger，避免增加额外配置成本。
- 完整测试用例接口暂未加服务间密钥，后续如需上线可增加内部 token。
- 性能没有做复杂缓存，分页查询和常用索引已满足课程项目规模。
