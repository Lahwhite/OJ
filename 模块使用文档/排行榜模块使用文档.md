# 排行榜模块使用文档

## 1. 模块概述

排行榜模块位于 `OJ/rank`，提供 OJ 系统的全球排名、竞赛排名、用户题目完成统计，以及 LeetCode 风格的可视化前端页面。

本模块实现以下能力：

1. 总排行榜（分页、多关键字排序）
2. 比赛排行榜（分页）
3. 用户题目完成统计（Easy / Medium / Hard）
4. 全局统计摘要、用户百分位洞察、CSV 导出
5. 排行榜 JSON / HTTP 接口
6. Windows 联调运行包

技术栈：

- 后端：C++17 + `common` 模块（`oj::HttpServer`、`oj::Router`、`OJ_LOG_*`、`loadConfigFromEnv`）
- 前端：`web/`（HTML + Tailwind CDN + ECharts）
- 存储：内存实现（可扩展 MySQL / Redis，见 `sql/leaderboard_schema.sql`）

模块主要对外提供两类入口：

- 前端页面：`GET /rank`
- REST API：`/api/leaderboard/...`

Users 模块只需在页面里增加跳转链接；本地联调可直接使用 `rank/runtime_package` 下的 Windows 运行包。

## 3. 核心组件

### 3.1 LeaderboardService

`LeaderboardService` 负责排行榜业务逻辑与排序。

主要能力：

- `get_global_leaderboard(limit, offset)`：获取总榜分页
- `get_contest_leaderboard(contest_id, limit, offset)`：获取比赛榜分页
- `get_problem_completion_stats(user_id)`：获取用户题目完成统计
- `on_submission(event)`：提交事件增量更新（含缓存失效）

### 3.2 LeaderboardApi

`LeaderboardApi` 将服务层结果序列化为 JSON 或 CSV，供 HTTP 层调用。

主要能力：

- `get_global_leaderboard_json(limit, offset)`
- `get_contest_leaderboard_json(contest_id, limit, offset)`
- `get_problem_stats_json(user_id)`
- `get_global_summary_json()`
- `get_user_insight_json(user_id)`
- `get_global_leaderboard_csv(limit, offset)`

### 3.3 LeaderboardHandler

`LeaderboardHandler` 基于 `common` 的 `oj::HttpServer` 注册路由，处理静态页面、API 与 `/health`。

当前已注册：

- `/`、`/rank`：前端页面（`/` 重定向到 `/rank`）
- `/rank/static/<path>`：前端静态资源
- `/api/leaderboard/global` 及 summary、export.csv
- `/api/leaderboard/contest/{contest_id}`
- `/api/leaderboard/users/{user_id}/stats`、`/insight`
- `/health`：健康检查（含 MySQL / Redis 状态）

### 3.4 StatsAnalyzer

`StatsAnalyzer` 提供统计摘要与导出：

- `analyze_global`：全局摘要（均值、中位数、分桶等）
- `analyze_user`：用户百分位洞察
- `rows_to_csv`：CSV 导出

默认分桶：

| 标签 | 分数区间 |
|------|----------|
| Bronze | 0 - 999 |
| Silver | 1000 - 1999 |
| Gold | 2000 - 2999 |
| Master | 3000+ |

## 4. 排名规则

总榜与比赛榜排序一致：

1. `score` 降序（主排序）
2. `solved_count` 降序
3. `penalty_seconds` 升序（罚时越低越好）
4. `last_accepted_at` 升序（更早达到者在前）

AC 得分权重：

| 难度 | 加分 |
|------|------|
| Easy | +100 |
| Medium | +220 |
| Hard | +400 |

同一用户同一题目在总榜只计一次 `solved_count`；竞赛榜按 `contest_id + user_id + problem_id` 去重。

## 5. 构建方式

排行榜模块依赖 `OJ/common`，CMake 会自动 `add_subdirectory(../common)`。

### 5.1 MSYS2 UCRT64（推荐）

```bash
pacman -S --needed mingw-w64-ucrt-x86_64-gcc mingw-w64-ucrt-x86_64-cmake mingw-w64-ucrt-x86_64-ninja

cd OJ/rank
cmake -G Ninja -DCMAKE_BUILD_TYPE=Release -DCMAKE_CXX_COMPILER=g++ -S . -B build-msys2
cmake --build build-msys2 --target leaderboard_server -j
```

### 5.2 运行测试

```bash
cmake --build build-msys2 --target leaderboard_tests leaderboard_extended_tests -j
./build-msys2/leaderboard_tests.exe
./build-msys2/leaderboard_extended_tests.exe

# 可选：common HTTP 冒烟测试
./build-msys2/oj_common_standalone/common_http_smoke_test.exe
```

## 6. 启动服务

### 6.1 开发模式

```bash
./build-msys2/leaderboard_server.exe
```

默认访问：

- 页面：`http://127.0.0.1:8092/rank`
- 健康检查：`http://127.0.0.1:8092/health`

### 6.2 Windows 运行包

`rank/runtime_package` 与 `discussion/runtime_package` 交付方式一致，包含 `leaderboard_server.exe`、MinGW 运行时 DLL、`web/`、`sql/` 与启动脚本。团队同学无需本机编译即可联调（当前为内存 demo 数据）。

在本目录运行：

```powershell
cd OJ\rank\runtime_package
powershell -ExecutionPolicy Bypass -File .\start_rank.ps1
```

### 6.3 环境变量

| 变量 | 说明 | 默认 |
|------|------|------|
| `OJ_RANK_PORT` | HTTP 端口 | `8092` |
| `OJ_LOG_LEVEL` | 日志级别 | `info` |
| `OJ_LOG_FILE` | 日志文件路径（可选） | 空 |
| `OJ_MYSQL_*` | MySQL 连接（`/health` 回报状态） | 见 `common` |
| `OJ_REDIS_*` | Redis 连接（`/health` 回报状态） | 见 `common` |

示例：

```powershell
$env:OJ_RANK_PORT="8093"
$env:OJ_LOG_LEVEL="info"
$env:OJ_LOG_FILE="rank.log"
powershell -ExecutionPolicy Bypass -File .\start_rank.ps1
```

日志与 HTTP 访问记录由 `common` 统一输出（如 `OJ_LOG_INFO`、`http GET /rank -> 200`）。

## 7. 同步运行包

修改 `rank` 源码或 `rank/web` 后，需要重新构建并同步到 `runtime_package`；旧运行包不会自动包含新接口或新页面。

推荐：

```powershell
powershell -ExecutionPolicy Bypass -File .\rank\runtime_package\sync_runtime.ps1
```

或手动：

```powershell
cmake --build rank\build-msys2 --target leaderboard_server -j
Copy-Item rank\build-msys2\leaderboard_server.exe rank\runtime_package\ -Force
Copy-Item rank\web\* rank\runtime_package\web\ -Force
```

说明：运行包为 **Windows + MSYS2 UCRT64** 构建；Linux/macOS 需对应平台重新 `cmake` 构建。

## 8. HTTP 接口说明

### 8.1 全球排行榜

- URL：`GET /api/leaderboard/global`
- 参数：`limit`（默认 50）、`offset`（默认 0）
- 返回：`{ "data": [ { "rank", "user_id", "username", "score", "solved_count", ... } ] }`

### 8.2 全球统计摘要

- URL：`GET /api/leaderboard/global/summary`
- 返回：`total_users`、`average_score`、`median_score`、`score_buckets` 等

### 8.3 用户排名洞察

- URL：`GET /api/leaderboard/users/{user_id}/insight`
- 返回：`rank`、`score_percentile`、`score_gap_to_top` 等

### 8.4 CSV 导出

- URL：`GET /api/leaderboard/global/export.csv`
- 参数：`limit`、`offset`
- 返回：`text/csv` 附件

### 8.5 竞赛排行榜

- URL：`GET /api/leaderboard/contest/{contest_id}`
- 参数：`limit`、`offset`

### 8.6 用户题目统计

- URL：`GET /api/leaderboard/users/{user_id}/stats`
- 返回：`solved_easy`、`solved_medium`、`solved_hard`、`solved_total`

### 8.7 健康检查

- URL：`GET /health`
- 返回：`status`、`service`、`mysql_pool_ready`、`redis_ready` 等

### 8.8 接口一览

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/rank` | 排行榜页面 |
| GET | `/api/leaderboard/global` | 总榜 JSON |
| GET | `/api/leaderboard/global/summary` | 全球统计摘要 |
| GET | `/api/leaderboard/global/export.csv` | 导出 CSV |
| GET | `/api/leaderboard/users/{id}/insight` | 用户百分位洞察 |
| GET | `/api/leaderboard/contest/{id}` | 比赛榜 JSON |
| GET | `/api/leaderboard/users/{id}/stats` | 用户题目完成统计 |
| GET | `/health` | 健康检查 |

## 9. 前端页面

访问 `/rank` 可见：

- 全球排名 / 竞赛排名 Tab
- Top3 领奖台卡片
- 排名表（点击行联动图表与用户统计）
- ECharts：积分渐变横条、难度堆叠柱、通过数+罚时双轴图
- 统计摘要条、导出 CSV 按钮

可选 URL 参数：

- `return_url`：返回主页地址
- `contest_id`：默认竞赛 ID
- `user_id`：默认选中用户

## 10. 与 Users 模块集成

Users 模块只需加链接，不需要编译 rank：

```html
<a th:href="@{http://127.0.0.1:8092/rank(return_url='http://127.0.0.1:8081/home')}">
    排行榜
</a>
```

生产环境将 `127.0.0.1` 替换为实际域名或网关地址。

## 11. 数据库与缓存（可选）

当前 `leaderboard_server` 默认使用**内存演示数据**，不依赖 MySQL。

若接入持久化：

1. 执行 `rank/sql/leaderboard_schema.sql`
2. 构建时开启 `ENABLE_MYSQL` / `ENABLE_REDIS`
3. 完善 `mysql_repository.cpp`、`redis_cache.cpp`

Redis 建议键设计：

- `global:zset`：总榜
- `contest:{id}:zset`：比赛榜
- `user:{id}:problem_stats`：题目统计

MySQL 建议表：

- `submissions`：事实表
- `leaderboard_global` / `leaderboard_contest` / `user_problem_stats`：聚合表

## 12. 常见问题

### 12.1 页面 404

确认访问 `http://127.0.0.1:8092/rank`（根路径 `/` 会重定向到 `/rank`）；服务进程需保持运行。

### 12.2 中文路径无法加载静态资源

已改为内存读取 `web/`，并支持从 exe 目录向上查找 `web/`。

### 12.3 端口占用

设置 `OJ_RANK_PORT=8093` 后重启。

### 12.4 运行包缺 DLL 或 exe 被占用

重新执行 `sync_runtime.ps1`；同步前先关闭正在运行的 `leaderboard_server.exe`。

### 12.5 与 discussion 的差异

排行榜已接入 `common` 的 HTTP 与日志，**不再使用 Crow**；联调方式与 discussion 的运行包一致。
