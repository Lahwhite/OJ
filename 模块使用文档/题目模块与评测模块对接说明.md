# 题目模块与评测模块（judge_engine）对接说明

本文说明 Problems 题目模块与 `judge_engine.exe` 评测引擎之间的数据对齐关系及推荐接入方式。

## 1. 模块职责划分

| 模块 | 职责 |
|---|---|
| **Problems（题目模块）** | 题目 CRUD、题面展示、测试用例存储、AC 状态存储与展示 |
| **judge_engine（评测引擎）** | 编译/运行用户代码、逐用例比对、输出评测结果 JSON |
| **提交/评测服务（待接入）** | 拉取题目数据、组装 CLI 参数、调用 judge_engine、解析结果、回写状态 |

Problems 模块**不参与代码评测**，只提供数据接口；详情页（`problem-detail.html`）的「运行」「提交」按钮为 UI 占位，实际评测由评测服务接入。

## 2. judge_engine CLI 命令格式

```text
judge_engine.exe --program_language=<语言> --src_file=<源码路径> --expect_result=<期望结果JSON路径>
```

参数说明（详见 `judge/使用文档.md`）：

- `--program_language`：编程语言，目前支持 `c` / `cpp` / `java` / `python`
- `--src_file`：用户源代码文件路径，引擎直接读取文件内容
- `--expect_result`：期望结果 JSON 文件路径，包含时间/内存限制和测试用例

## 3. 字段对齐关系

题目模块存储的数据结构与 `expect_result` JSON **已对齐**，接入时做字段名映射即可。

### 3.1 限制字段

| expect_result | Problems API | 数据库字段 |
|---|---|---|
| `time_limit_ms` | `timeLimit` | `problems.time_limit` |
| `memory_limit_mb` | `memoryLimit` | `problems.memory_limit` |

数据来源：`GET /problems/v1/problems/{id}`

### 3.2 测试用例字段

| expect_result | Problems API | 数据库字段 |
|---|---|---|
| `test_cases[].id` | `id` | `test_cases.id` |
| `test_cases[].input` | `input` | `test_cases.input` |
| `test_cases[].expected_output` | `output` | `test_cases.output` |
| `test_cases[].score` | `score` | `test_cases.score` |

数据来源：`GET /problems/v1/problems/{id}/test-cases`

> judge_engine 同时接受 `expected_output` 与 `output` 作为期望输出字段名；`test_cases` 与 `testcases` 作为用例数组字段名均可。

### 3.3 语言参数

| 详情页语言选择 | `--program_language` |
|---|---|
| `c` | `c` |
| `cpp` | `cpp` |
| `java` | `java` |
| `python` | `python` |
| `javascript` | **不支持**（judge_engine 无此语言，接入时需拦截） |

## 4. expect_result 组装示例

假设 `GET /problems/v1/problems/1` 返回：

```json
{
  "id": 1,
  "title": "两数之和",
  "timeLimit": 1000,
  "memoryLimit": 128
}
```

`GET /problems/v1/problems/1/test-cases` 返回：

```json
[
  { "id": 10, "input": "1 2", "output": "3", "isSample": true, "score": 50 },
  { "id": 11, "input": "10 20", "output": "30", "isSample": false, "score": 50 }
]
```

组装后的 `expect_result.json`：

```json
{
  "time_limit_ms": 1000,
  "memory_limit_mb": 128,
  "test_cases": [
    {
      "id": 10,
      "input": "1 2",
      "expected_output": "3",
      "score": 50
    },
    {
      "id": 11,
      "input": "10 20",
      "expected_output": "30",
      "score": 50
    }
  ]
}
```

## 5. 已实现提交接口

Problems 模块已提供对接接口：

```http
POST /problems/v1/problems/{id}/submit
Content-Type: application/json
```

```json
{
  "username": "admin",
  "language": "java",
  "code": "public class Main { public static void main(String[] args) {} }"
}
```

后端流程：

1. 读取题目 `timeLimit` / `memoryLimit` 和 `test_cases`
2. 在 `judge/` 目录写入源码与 `{prefix}_cases.json`
3. 执行 `judge_engine.exe --program_language=... --src_file=... --expect_result=... --username=...`
4. 返回简单提交结果（详细 verdict 由 judge 模块生成结果 JSON 后处理）

前端要求 URL 带 `username`，例如：

```text
/problems/?username=admin&return_url=...
/problem-detail.html?id=1&username=admin
```

详情页「运行代码」已隐藏；「提交答案」调用上述接口。

配置（`Problems/src/main/resources/application.yml`）：

- `judge.work-dir`：默认 `../judge`（按 `Problems` 模块相对路径定位到项目根目录下的 `judge/`）
- `judge.executable`：默认 `judge_engine.exe`

## 6. 推荐接入流程（历史说明）

```text
┌─────────────┐     提交请求      ┌──────────────┐
│  详情页/UI  │ ───────────────→ │ 提交/评测服务 │
└─────────────┘                  └──────┬───────┘
                                        │
                    GET /problems/v1/problems/{id}
                    GET /problems/v1/problems/{id}/test-cases
                                        │
                                        ▼
                              组装 expect_result.json
                              写入用户源码 → src 文件
                                        │
                                        ▼
                    judge_engine.exe --program_language=...
                                     --src_file=...
                                     --expect_result=...
                                        │
                                        ▼
                              读取评测结果 JSON
                              (verdict, total_score, max_score, ...)
                                        │
                    PUT /problems/v1/problem-status
                                        │
                                        ▼
                              返回结果给前端展示
```

## 7. 评测完成后回写 AC 状态

评测服务在得到 judge_engine 返回结果后，调用 Problems 模块的状态接口：

```http
PUT /problems/v1/problem-status
X-Service-Token: <OJ_PROBLEM_STATUS_TOKEN>
Content-Type: application/json
```

```json
{
  "userId": 1,
  "problemId": 1,
  "accepted": true,
  "score": 100,
  "maxScore": 100,
  "submittedAt": "2026-06-05T12:00:00"
}
```

字段映射建议：

| 状态接口字段 | judge_engine 返回 |
|---|---|
| `accepted` | `verdict == "AC"` |
| `score` | `total_score` |
| `maxScore` | `max_score` |

## 8. judge_engine 返回结果要点

CLI 执行后输出 JSON（详见 `judge/使用文档.md` §3.3）：

- `verdict`：最终结果（`AC` / `WA` / `TLE` / `MLE` / `RE` / `CE`）
- `total_score` / `max_score`：得分
- `runtime` / `memory`：运行时间与内存
- `test_cases[]`：每个用例的 `case_id`、`verdict`、`passed`、`score` 等
