# 排行榜模块使用文档

## 1. 模块概述

排行榜模块（`OJ/rank`）负责 OJ 系统的全球排名、竞赛排名、用户题目完成统计，以及 LeetCode 风格的可视化前端页面。

核心能力：

1. 总排行榜（分页、多关键字排序）
2. 竞赛排行榜（分页）
3. 用户题目完成统计（Easy / Medium / Hard）
4. 全局统计摘要（均值、分桶、百分位）
5. CSV 导出
6. Windows 运行包一键启动

技术栈：

- 后端：C++17 + Crow HTTP
- 前端：HTML + Tailwind CDN + ECharts
- 存储：内存实现（可扩展 MySQL / Redis）

## 3. 排名规则

总榜与比赛榜排序一致：

1. `score` 降序
2. `solved_count` 降序
3. `penalty_seconds` 升序
4. `last_accepted_at` 升序

AC 得分权重：

| 难度 | 加分 |
|------|------|
| Easy | +100 |
| Medium | +220 |
| Hard | +400 |

同一用户同一题目在总榜只计一次 `solved_count`；竞赛榜按 `contest_id + user_id + problem_id` 去重。

## 4. 构建方式

### 4.1 MSYS2 UCRT64（推荐）

```bash
pacman -S --needed mingw-w64-ucrt-x86_64-gcc mingw-w64-ucrt-x86_64-cmake mingw-w64-ucrt-x86_64-ninja
cd OJ/rank
cmake -G Ninja -DCMAKE_BUILD_TYPE=Release -DCMAKE_CXX_COMPILER=g++ -S . -B build-msys2
cmake --build build-msys2 --target leaderboard_server -j
```

### 4.2 运行测试

```bash
cmake --build build-msys2 --target leaderboard_tests -j
cmake --build build-msys2 --target leaderboard_extended_tests -j
./build-msys2/leaderboard_tests.exe
./build-msys2/leaderboard_extended_tests.exe
```

## 5. 启动服务

### 5.1 开发模式

```bash
./build-msys2/leaderboard_server.exe
```

默认地址：`http://127.0.0.1:8092/rank`

### 5.2 Windows 运行包

```powershell
cd OJ\rank\runtime_package
powershell -ExecutionPolicy Bypass -File .\start_rank.ps1
```

同步运行包：

```powershell
powershell -ExecutionPolicy Bypass -File .\rank\runtime_package\sync_runtime.ps1
```

## 6. HTTP 接口

### 6.1 全球排行榜

- URL：`GET /api/leaderboard/global`
- 参数：`limit`、`offset`
- 返回：`{ "data": [ { "rank", "user_id", "username", "score", ... } ] }`

### 6.2 全球统计摘要

- URL：`GET /api/leaderboard/global/summary`
- 返回：总人数、均值、中位数、分桶人数、最高分差距等

### 6.3 用户排名洞察

- URL：`GET /api/leaderboard/users/{id}/insight`
- 返回：名次、百分位、与榜首分差

### 6.4 CSV 导出

- URL：`GET /api/leaderboard/global/export.csv`
- 参数：`limit`、`offset`
- 返回：`text/csv` 附件

### 6.5 竞赛排行榜

- URL：`GET /api/leaderboard/contest/{contest_id}`
- 参数：`limit`、`offset`

### 6.6 用户题目统计

- URL：`GET /api/leaderboard/users/{id}/stats`
- 返回：`solved_easy`、`solved_medium`、`solved_hard`

### 6.7 健康检查

- URL：`GET /health`

## 7. 前端页面

访问 `/rank` 可见：

- 全球 / 竞赛 Tab
- Top3 领奖台
- 排名表（点击联动图表）
- ECharts：积分横条、难度堆叠柱、通过数+罚时双轴图
- 统计摘要条（调用 `/api/leaderboard/global/summary`）
- 导出 CSV 按钮

可选 URL 参数：

- `return_url`：返回链接
- `contest_id`：默认竞赛 ID
- `user_id`：默认选中用户

## 8. 与 Users 模块集成

在 `home.html` 增加链接：

```html
<a th:href="@{http://127.0.0.1:8092/rank(return_url='http://127.0.0.1:8081/home')}">
    排行榜
</a>
```

生产环境将 `127.0.0.1` 替换为实际域名或网关地址。

## 9. 数据库（可选）

当前 `leaderboard_server` 默认使用内存演示数据。若接入 MySQL：

1. 执行 `sql/leaderboard_schema.sql`
2. 启用 `ENABLE_MYSQL` 构建
3. 实现 `mysql_repository.cpp` 中的真实查询与写入

Redis 缓存适配位于 `redis_cache.cpp`，用于热点排行榜缓存。

## 10. 统计模块说明

`StatsAnalyzer` 提供：

- `analyze_global`：全局摘要
- `analyze_user`：用户百分位洞察
- `rows_to_csv`：导出
- `summary_to_json` / `insight_to_json`：API 序列化

分桶默认规则：

| 标签 | 分数区间 |
|------|----------|
| Bronze | 0 - 999 |
| Silver | 1000 - 1999 |
| Gold | 2000 - 2999 |
| Master | 3000+ |

## 11. 常见问题

### 11.1 页面 404

确认访问 `/rank` 而非根路径；服务进程需保持运行。

### 11.2 中文路径无法加载静态资源

已改为内存读取 `web/`，并支持从 exe 目录向上查找 `web`。

### 11.3 端口占用

设置 `OJ_RANK_PORT=8093` 后重启。

### 11.4 运行包缺 DLL

重新执行 `sync_runtime.ps1`，确保 `C:\msys64\ucrt64\bin` 下运行时 DLL 被复制。
