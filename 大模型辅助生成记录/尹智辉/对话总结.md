## 遇到的问题

1. **跨平台命令兼容性**
   - 初始使用Linux风格的`mkdir -p`命令创建目录，在Windows PowerShell中失败
   - 错误信息："标记'&&'不是此版本中的有效语句分隔符"

2. **沙箱环境实现**
   - 开发指南中使用Linux系统调用实现沙箱，在Windows环境中不兼容
   - 需要适配Windows API来实现资源限制和进程隔离

3. **依赖管理**
   - 项目依赖多个第三方库（Crow、MySQL Connector/C++、RabbitMQ等）
   - 需要在CMakeLists.txt中正确配置依赖项

## 决策与理由

1. **使用PowerShell兼容命令**
   - 决策：使用`New-Item -ItemType Directory -Force`命令创建目录
   - 理由：确保在Windows环境中能够正确创建目录结构，保证跨平台兼容性

2. **适配Windows沙箱实现**
   - 决策：使用Windows API（CreateProcess、WaitForSingleObject等）实现沙箱
   - 理由：在Windows环境中提供安全的代码执行环境，确保评测过程的安全性

3. **模块化设计**
   - 决策：采用模块化设计，将评测模块分为编译器、沙箱、结果判定器等组件
   - 理由：提高代码可维护性和可扩展性，便于测试和调试

4. **多语言支持**
   - 决策：实现对C、C++、Java、Python四种语言的支持
   - 理由：满足OJ系统的核心需求，支持常见的编程竞赛语言

5. **HTTP接口设计**
   - 决策：使用Crow框架实现RESTful API
   - 理由：提供简洁的接口用于代码提交和查询，便于与前端集成
