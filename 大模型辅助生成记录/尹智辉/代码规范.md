# OJ系统代码规范

## 1. 命名规范

### 1.1 文件命名
- 头文件：使用`.h`后缀，文件名使用小写字母，单词之间用下划线分隔，如`compiler.h`
- 源文件：使用`.cpp`后缀，文件名与头文件对应，如`compiler.cpp`
- 测试文件：使用`test_`前缀，如`test_compiler.cpp`

### 1.2 变量命名
- 局部变量：使用小驼峰命名法，如`sourceCode`
- 成员变量：使用小驼峰命名法，后缀加下划线，如`config_`
- 常量：使用全大写，单词之间用下划线分隔，如`MAX_MEMORY_LIMIT`

### 1.3 函数命名
- 使用小驼峰命名法，如`compileCode`
- 函数名应清晰表达函数的功能

### 1.4 类命名
- 使用大驼峰命名法，如`Compiler`
- 类名应清晰表达类的职责

## 2. 代码格式

### 2.1 缩进
- 使用4个空格进行缩进，不使用制表符
- 函数体、循环体、条件语句等都要缩进

### 2.2 括号
- 大括号使用独占一行的风格
- 左大括号放在行尾，右大括号放在新行的开始

### 2.3 空格
- 操作符两边加空格，如`a = b + c`
- 逗号后加空格，如`function(a, b, c)`
- 分号后加空格（如果在同一行）

### 2.4 行长度
- 每行代码长度不超过80个字符
- 超过80个字符的代码应换行，换行后的代码应适当缩进

## 3. 注释规范

### 3.1 文件头部注释
- 每个文件都应有头部注释，包括文件描述、作者、创建日期等

### 3.2 类注释
- 每个类都应有注释，描述类的职责、使用方法等

### 3.3 函数注释
- 每个函数都应有注释，包括函数功能、参数说明、返回值说明等
- 使用Doxygen格式的注释

### 3.4 代码注释
- 复杂的代码块应有注释，说明代码的逻辑
- 关键变量和常量应有注释，说明其用途

## 4. 错误处理

- 应使用异常处理机制处理错误
- 错误信息应清晰明了，便于调试
- 应记录错误日志，便于问题排查

## 5. 性能优化

- 避免不必要的内存分配和复制
- 使用适当的数据结构和算法
- 考虑并发处理，提高系统性能

## 6. 安全注意事项

- 防止代码注入攻击
- 限制资源使用，防止资源耗尽攻击
- 隔离用户代码，防止恶意代码执行

## 7. 示例代码

### 7.1 头文件示例
```cpp
/**
 * @file compiler.h
 * @brief 编译器接口定义
 * @author OJ Team
 * @date 2024-01-01
 */
#ifndef COMPILER_H
#define COMPILER_H

#include <string>
#include <vector>

/**
 * @struct CompileResult
 * @brief 编译结果结构体
 */
struct CompileResult {
    bool success;        ///< 编译是否成功
    std::string output;  ///< 编译输出
    std::string error;   ///< 编译错误信息
    int exit_code;       ///< 编译退出码
};

/**
 * @class Compiler
 * @brief 编译器基类
 */
class Compiler {
public:
    /**
     * @brief 析构函数
     */
    virtual ~Compiler() = default;
    
    /**
     * @brief 编译代码
     * @param source_code 源代码
     * @param output_path 输出文件路径
     * @return 编译结果
     */
    virtual CompileResult compile(const std::string& source_code, 
                            const std::string& output_path) = 0;
};

#endif // COMPILER_H
```

### 7.2 源文件示例
```cpp
/**
 * @file compiler.cpp
 * @brief 编译器实现
 * @author OJ Team
 * @date 2024-01-01
 */
#include "compiler.h"
#include <iostream>
#include <fstream>
#include <cstdio>
#include <string>
#include <sstream>

/**
 * @brief 执行命令并获取输出
 * @param command 命令字符串
 * @return 执行结果
 */
CompileResult executeCommand(const std::string& command) {
    CompileResult result;
    result.success = false;
    result.exit_code = -1;
    
    FILE* pipe = popen(command.c_str(), "r");
    if (!pipe) {
        result.error = "Failed to open pipe";
        return result;
    }
    
    char buffer[1024];
    std::string output;
    
    while (fgets(buffer, sizeof(buffer), pipe) != nullptr) {
        output += buffer;
    }
    
    result.exit_code = pclose(pipe);
    result.success = (result.exit_code == 0);
    result.output = output;
    
    return result;
}

/**
 * @brief 保存代码到临时文件
 * @param code 代码内容
 * @param extension 文件扩展名
 * @return 临时文件路径
 */
std::string saveToTempFile(const std::string& code, const std::string& extension) {
    char temp_name[L_tmpnam_s + extension.size() + 1];
    tmpnam_s(temp_name, L_tmpnam_s);
    std::string temp_file = std::string(temp_name) + extension;
    
    std::ofstream file(temp_file);
    if (file.is_open()) {
        file << code;
        file.close();
    }
    
    return temp_file;
}

/**
 * @brief 编译C代码
 * @param source_code C源代码
 * @param output_path 输出文件路径
 * @return 编译结果
 */
CompileResult CCompiler::compile(const std::string& source_code, const std::string& output_path) {
    std::string temp_file = saveToTempFile(source_code, ".c");
    std::string command = "gcc -std=c11 -O2 -o " + output_path + " " + temp_file;
    
    CompileResult result = executeCommand(command);
    
    // 清理临时文件
    remove(temp_file.c_str());
    
    return result;
}
```

## 8. 代码审查

- 定期进行代码审查，确保代码符合规范
- 代码审查应包括代码格式、注释、错误处理等方面
- 代码审查应记录问题和改进建议
