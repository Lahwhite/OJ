# 代码评测模块开发指南

## 一、模块概述

### 1.1 职责
代码评测模块是OJ系统的核心模块，负责：
- 接收用户代码提交
- 编译用户代码
- 在安全沙箱环境中运行代码
- 判定评测结果
- 返回评测详情

### 1.2 技术栈
- **语言**: C++17
- **HTTP框架**: Crow
- **数据库**: MySQL Connector/C++
- **消息队列**: RabbitMQ (AMQP-CPP)
- **容器化**: Docker
- **JSON**: nlohmann/json

### 1.3 依赖模块
- 公共服务模块：HTTP服务器、数据库连接池、日志系统
- 题目管理模块：获取题目信息和测试用例

## 四、核心功能实现要点

### 4.1 编译流程

1. 接收源代码
2. 保存到临时文件
3. 执行编译命令
4. 检查编译结果
5. 返回编译结果

### 4.2 沙箱安全

**资源限制**：
```cpp
bool Sandbox::setupResourceLimits() {
    struct rlimit time_limit;
    time_limit.rlim_cur = config_.time_limit_ms / 1000;
    time_limit.rlim_max = config_.time_limit_ms / 1000;
    
    if (setrlimit(RLIMIT_CPU, &time_limit) != 0) {
        return false;
    }
    
    struct rlimit memory_limit;
    memory_limit.rlim_cur = config_.memory_limit_mb * 1024 * 1024;
    memory_limit.rlim_max = config_.memory_limit_mb * 1024 * 1024;
    
    if (setrlimit(RLIMIT_AS, &memory_limit) != 0) {
        return false;
    }
    
    return true;
}
```

**进程隔离**：
- 使用 `fork()` 创建子进程
- 使用 `chroot()` 限制文件系统访问
- 使用 `setuid()` 降低权限

### 4.3 结果判定

```cpp
JudgeStatus ResultJudger::determineStatus(
    const SandboxResult& result,
    bool output_matches) {
    
    if (result.timeout) {
        return JudgeStatus::TIME_LIMIT_EXCEEDED;
    }
    
    if (result.memory_exceeded) {
        return JudgeStatus::MEMORY_LIMIT_EXCEEDED;
    }
    
    if (result.exit_code != 0) {
        return JudgeStatus::RUNTIME_ERROR;
    }
    
    if (!output_matches) {
        return JudgeStatus::WRONG_ANSWER;
    }
    
    return JudgeStatus::ACCEPTED;
}
```

### 4.4 输出比较

```cpp
bool ResultJudger::compareOutput(const std::string& actual,
                            const std::string& expected) {
    std::string actual_trimmed = trim(actual);
    std::string expected_trimmed = trim(expected);
    
    return actual_trimmed == expected_trimmed;
}

std::string trim(const std::string& str) {
    size_t first = str.find_first_not_of(" \t\n\r");
    if (std::string::npos == first) return str;
    
    size_t last = str.find_last_not_of(" \t\n\r");
    return str.substr(first, (last - first + 1));
}
```

## 七、测试用例

### 编译器测试
```cpp
TEST(CompilerTest, CompileValidCppCode) {
    CppCompiler compiler;
    std::string code = "#include <iostream>\nint main() { return 0; }";
    CompileResult result = compiler.compile(code, "/tmp/test.out");
    EXPECT_TRUE(result.success);
    EXPECT_EQ(result.exit_code, 0);
}

TEST(CompilerTest, CompileInvalidCppCode) {
    CppCompiler compiler;
    std::string code = "invalid code";
    CompileResult result = compiler.compile(code, "/tmp/test.out");
    EXPECT_FALSE(result.success);
    EXPECT_FALSE(result.error.empty());
}
```

### 沙箱测试
```cpp
TEST(SandboxTest, ExecuteWithinTimeLimit) {
    SandboxConfig config;
    config.time_limit_ms = 1000;
    config.memory_limit_mb = 128;
    
    Sandbox sandbox(config);
    SandboxResult result = sandbox.execute("./test", "test input");
    
    EXPECT_FALSE(result.timeout);
    EXPECT_LE(result.runtime_ms, 1000);
}

TEST(SandboxTest, ExecuteExceedsTimeLimit) {
    SandboxConfig config;
    config.time_limit_ms = 100;
    config.memory_limit_mb = 128;
    
    Sandbox sandbox(config);
    std::string code = "while(1) {}";
    SandboxResult result = sandbox.execute("./infinite_loop", "");
    
    EXPECT_TRUE(result.timeout);
}
```

## 八、注意事项

### 安全注意事项
1. **代码注入防护**：严格验证输入，防止命令注入
2. **资源限制**：设置合理的CPU和内存限制
3. **文件隔离**：使用chroot或容器隔离
4. **权限控制**：使用最小权限原则

### 性能注意事项
1. **并发评测**：使用线程池处理多个提交
2. **缓存优化**：缓存编译结果和测试用例
